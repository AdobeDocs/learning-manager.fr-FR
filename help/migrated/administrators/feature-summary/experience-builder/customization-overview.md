# Vue d’ensemble

Experience Builder permet aux administrateurs de créer des expériences d’apprentissage personnalisées au sein de la plateforme, en comblant le fossé entre les interfaces natives prêtes à l’emploi et les solutions sans tête. Il permet de concevoir des pages personnalisées à l’aide de divers widgets, de configurations de menu flexibles et d’un branding avancé via CSS et JavaScript. Cette approche réduit les délais de commercialisation et les coûts de mise en œuvre par rapport aux versions sans en-tête, tout en offrant une personnalisation approfondie pour l’image de marque et la mise en page.

## Ce que vous pouvez personnaliser

**Pied de page**

Vous avez deux options pour le pied de page. Vous pouvez utiliser le pied de page par défaut, qui est configuré dans les paramètres d’administration généraux, ou vous pouvez sélectionner un pied de page personnalisé. L’option personnalisée vous permet de saisir votre propre HTML et CSS, ce qui vous donne un contrôle total sur la conception et le contenu du pied de page en fonction de votre marque.

Consultez cette section pour plus d’informations.

**Configuration globale CSS et JS**

Personnalisez l’apparence et les fonctionnalités de l’application de l’élève en injectant des feuilles de style CSS et JavaScript personnalisées. Cela permet un branding précis, des modifications dynamiques de l’interface utilisateur et des fonctionnalités interactives avancées. En outre, personnalisez l’apparence et les fonctionnalités de l’application de l’élève en injectant des feuilles de style CSS et JavaScript personnalisées. Cela permet un branding précis, des modifications dynamiques de l’interface utilisateur et des fonctionnalités interactives avancées.

Consultez cette section pour plus d’informations.

**Vignette de cours**

Configurez globalement les détails qui apparaissent sur les cartes d’objet d’apprentissage dans l’ensemble du compte. Vous pouvez afficher ou masquer des éléments, tels que le nom de l’auteur, le nom de la compétence, la note, la description et le bouton d’inscription. Toutefois, les métadonnées de base telles que le titre, l’image, le prix et la barre de progression ne peuvent pas être supprimées.

Consultez cette section pour plus d’informations.

**Widgets**

Personnalisez les widgets dans Experience Builder en effectuant une sélection parmi divers types de widgets (par exemple, calendrier, catégories, cours et parcours, zone de contenu personnalisée, ludification, HTML, iFrame, apprentissage par les réseaux sociaux, mon apprentissage) et en configurant leurs propriétés, leur mise en page et leur contenu. Les widgets prennent en charge les titres personnalisés, les descriptions, les images, les actions et, dans certains cas, les options avancées telles que l’injection de HTML, CSS et JavaScript pour des expériences dynamiques ou pilotées par l’API.

Consultez cette section pour plus d’informations.

## Cas d’utilisation

**Créez des portails d’apprentissage personnalisés basés sur les rôles**

Proposez différentes expériences à différents élèves au sein d&#39;un même compte, telles que la création de pages et de menus personnalisés ou d&#39;une bande d&#39;apprentissage organisée pour un public particulier.

En outre, désignez une page personnalisée publiée comme page de destination pour un groupe d’utilisateurs spécifique. Cela garantit que lorsque les élèves de ce groupe se connectent, ils sont immédiatement redirigés vers une expérience pertinente et personnalisée au lieu de la page d’accueil par défaut.

**Création de pages d’apprentissage spécifiques à l’événement**

Experience Builder est idéal pour créer des expériences d’apprentissage temporaires et ciblées pour les événements d’entreprise ou les initiatives sensibles au facteur temps.

* **Événements d’entreprise** : pour un événement tel qu’un « Design Summit », un administrateur peut créer de nouvelles pages spécifiquement pour l’équipe de conception. Ces pages peuvent inclure des informations sur les sessions, des listes d’intervenants et des supports de formation préalables aux événements. Le widget Calendrier est particulièrement utile ici, car il peut être ajouté à la page pour afficher les sessions à venir et permettre aux participants de s’inscrire et de les suivre.
* **Campagnes limitées dans le temps** : créez une campagne temporaire, telle qu&#39;un « coup d&#39;envoi des ventes » de deux mois. Pour cette période, une page d’accueil et un menu personnalisés ne peuvent être rendus visibles que par l’équipe commerciale, avec une bannière comme « Commencer les ventes 2026 » et des cours spécifiques à leur stratégie pour le nouvel exercice. Après deux mois, le portail peut revenir à l’expérience standard pour tout le monde. Cela permet la création de « campagnes de micro-apprentissage » ciblées.

**Développer des académies clients de marque**

Experience Builder permet aux entreprises de créer des portails de formation client sur la marque sans les coûts élevés et le long temps de développement d’une mise en œuvre entièrement sans interface utilisateur.

Pour une flexibilité maximale, le widget HTML, par exemple, vous permet d’incorporer du code HTML, CSS et JS personnalisé pour des éléments tels que des témoignages ou des bannières au style unique. Cela vous donne plus de contrôle sur l’expérience de branding.

**Intégrer du contenu personnalisé**

Configurez les menus pour qu’ils apparaissent à différents emplacements (rail de gauche, navigation supérieure, en-tête inférieur) et contrôlez la visibilité de la page pour des parcours d’utilisateurs rationalisés.

* **Créer des sections de contenu enrichies** : le widget Zone de contenu vous permet de créer facilement des sections avec des titres, des descriptions, des images et des appels à l&#39;action (CTA) pour guider les élèves ou mettre en évidence des informations clés.
* **Encourager la collaboration et la concurrence** : vous pouvez ajouter le widget Apprentissage par les réseaux sociaux pour afficher les publications et les commentaires, ou le widget Ludification pour afficher les tableaux des scores et les points directement sur une page personnalisée, ce qui stimule l’engagement.
* **Organiser le contenu pour faciliter la navigation** : utilisez le widget Catégories pour afficher des catalogues, des produits ou des rôles sous la forme d&#39;une bande de vignettes, afin de fournir aux élèves des chemins de navigation clairs et visuels vers différentes zones de contenu.
Intégration d’outils externes
* **Intégrer des applications externes** : le widget iFrame vous permet d’intégrer des applications ou des pages web externes directement dans une page personnalisée, créant ainsi une expérience plus fluide pour l’élève.
* **Créer des composants dynamiques avec des API** : le widget de HTML vous permet d&#39;ajouter du code JavaScript pour permettre aux utilisateurs d&#39;interagir avec, par exemple, les API des élèves. Grâce à cette approche, vous pouvez créer une expérience dans l’environnement Learning Manager natif sans avoir à créer une nouvelle application.
