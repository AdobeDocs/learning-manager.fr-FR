# Gestion des commandes et de la facturation Learning Manager

L’achat par carte de crédit est uniquement disponible dans la section [Région États-Unis](http://learningmanager.adobe.com/).

Gérez la facturation Learning Manager, passez des commandes en utilisant une carte de crédit, abonnez-vous en utilisant un bon de commande, ou via une formule Utilisateurs actifs mensuels.

Adobe Learning Manager propose un modèle de tarification flexible, convivial et l’un des meilleurs modèles de tarification pour répondre aux besoins de votre organisation. Pour plus d’informations, voir la section [Learning Manager](https://www.adobe.com/products/learningmanager.html) page.

Seuls les administrateurs de votre organisation peuvent gérer la facturation.

Si vous souhaitez contacter l’Adobe pour plus d’informations sur l’abonnement Learning Manager et la facturation, écrivez-nous à l’adresse [learningmanagersales@adobe.com](mailto:learningmanagersales@adobe.com).

## Passer des commandes à l’aide de cartes de crédit {#placeordersusingcreditcards}

Vous pouvez acheter un abonnement pour un maximum de 3 500 élèves par le biais d’un seul ordre de paiement par carte de crédit. La première commande du compte doit être destinée à au moins 10 élèves.

1. Dans l’application Administrateur, cliquez sur **[!UICONTROL Facturation]** dans le volet de navigation de gauche.

   ![](assets/billing.png)

   *Lancer la facturation Adobe Learning Manager*

1. Sur la **[!UICONTROL Informations de facturation]** ajouter le nombre d&#39;utilisateurs dans la page **[!UICONTROL Ajouter des utilisateurs]** champ. Lorsque vous utilisez une carte de crédit pour des abonnements prépayés, vous pouvez voir le nombre d’utilisateurs que vous pouvez ajouter pour l’abonnement. Le nombre d’utilisateurs que vous pouvez ajouter ne doit pas dépasser le nombre mentionné dans la section Restant.1.

   ![](assets/billing-page-to-manageyoursubscriptionandorders.png)

   *Ajouter un nombre d’utilisateurs*

1. Après avoir spécifié le nombre d’utilisateurs à ajouter, cliquez sur Passer une commande dans le coin supérieur droit de la page.

   ![](assets/billing2.png)

1. Vérifiez le devis affiché à l’écran.

   ![](assets/pricing-estimate.png)

   *Passer une commande*

   Les frais d’abonnement annuel sont calculés en fonction du nombre d’utilisateurs ajoutés à l’abonnement. Par exemple, si quatre utilisateurs sont ajoutés, les frais annuels sont calculés à l’aide de l’expression 4 utilisateursX$4 X$12, qui renvoie 192 $.

   Cliquez sur **[!UICONTROL Continuer]**.

   *Consulter l’estimation*

   ![](assets/payment-details.png)

## Vérifier le statut de la commande {#checkorderstatus}

Toutes les commandes peuvent avoir l’un des quatre statuts suivants :

**Actif :** Une commande est active et les utilisateurs sont enregistrés.

**Suspendu :** Un ordre passe à l’état suspendu dans les scénarios suivants :

* Retard de réception du paiement de la carte bancaire
* Expiration de la carte bancaire.
* Le paiement est refusé pour tout cycle de paiement récurrent.

**Annulation initiée :** Un ordre passe dans cet état lorsque l’administrateur de Learning Manager désactive le compte. La commande passe ensuite dans un état annulé après réception de la confirmation d&#39;annulation de la commande.

## Mettre à jour les détails de l’abonnement {#updatesubscriptiondetails}

1. Dans la liste des commandes, cliquez sur **[!UICONTROL Modifier]**.

   ![](assets/update-subsciptiondetailsclickedit.png)

   *Mettre à jour les détails de l’abonnement*

1. Dans la page Détails de l’abonnement, cliquez sur **[!UICONTROL Modifier l’abonnement]**.
1. Choisissez l’élément à modifier :

   * Mode de paiement : utilisez cette option pour mettre à jour les détails de paiement, tels que les données de carte bancaire.
   * Adresse : utilisez cette option pour mettre à jour les détails de l’adresse.

## Résiliation d’un abonnement {#cancelasubscription}

Pour annuler une commande :

1. Dans le volet gauche de la page Administrateur, cliquez sur Facturation.
1. Dans la page Facturation, dans le coin supérieur droit, choisissez **[!UICONTROL Actions]** > **[!UICONTROL Désactiver le compte]**.
1. Une fois que l’administrateur a désactivé le compte, toutes les commandes existantes dans le compte sont annulées à partir du cycle de facturation suivant.

Lorsqu’un compte est désactivé par le client, il passe à l’état d’évaluation pendant les 30 jours suivants. Le propriétaire du compte reçoit trois e-mails de rappel pour réactiver le compte. Si le propriétaire ne réactive pas le compte, aucun des utilisateurs ne peut accéder à Learning Manager à part lui.

## Passer des commandes à l&#39;aide de la commande {#placeordersusingpurchaseorder}

Vous pouvez choisir le processus de commande fournisseur comme mode de paiement alternatif. Comme condition préalable, le compte de votre organisation doit être enregistré auprès d’Adobe. Le compte de votre organisation est facturé pour ce processus. Le compte est facturé en fonction des activités d’un élève. Seules les activités au niveau de l’objet d’apprentissage sont facturées. Pour passer une commande à l&#39;aide de PO :

1. Envoyer un e-mail à [learningmanagersales@adobe.com](mailto:learningmanagersales@adobe.com) et mentionnez le nombre d’élèves requis.
1. L’équipe Learning Manager vous envoie une clé d’activation.
1. Dans la page Facturation de l’application Administrateur, saisissez la clé d’activation.
1. Cliquez sur Activer dans le coin supérieur droit de la page.

## Vérifier l’état du compte {#checkaccountstatus}

Une fois le compte activé, il peut présenter l’un des états suivants :

* **Évaluation** - Vous pouvez créer un compte Learning Manager Adobe et l’utiliser sans frais pendant une période de 30 jours. Il n’y a pas de limite au nombre d’élèves inscrits pendant la période d’essai.
* **Actif** - Dans cet état, le compte a des abonnements d’élève actifs avec un paiement mensuel récurrent conformément à la commande d’abonnement.
* **Inactif** - Un compte passe à l’état inactif dans les scénarios suivants :

   * Après la période d’essai s’il n’y a aucune commande d’abonnement active dans le compte.
   * L’administrateur désactive le compte, ce qui entraîne l’annulation de toutes les commandes existantes dans un compte à partir du prochain cycle de facturation de l’abonnement.
   * Le paiement est refusé pour les commandes actives dans un compte même après les rappels.

Un état inactif n’annule pas votre compte avec effet immédiat. Vous recevez au moins deux rappels de la part de l’équipe de Learning Manager vous demandant de fournir les dernières informations sur

votre carte de crédit si elle a expiré. Dans un état inactif, seul un administrateur peut se connecter au Captivate

Compte Learning Manager. Aucun autre utilisateur ne peut accéder au compte.

* **Activation requise** - Votre compte passe dans cet état lorsque l’administrateur de Learning Manager choisit de désactiver le compte. Toutes les commandes de ce compte sont annulées. Le paiement de ces commandes ne se fait pas à partir du prochain cycle de facturation. Le compte conserve cet état jusqu’au jour du dernier cycle de facturation. Dans ce cas, tous les utilisateurs peuvent continuer à utiliser l’application sans aucun impact jusqu’à la fin de la dernière date de paiement récurrente.

## Frais de résiliation du compte {#accountterminationfee}

Si vous souhaitez annuler l’abonnement avant la fin de la période annuelle, des frais de résiliation anticipée sont facturés. Les frais de résiliation sont équivalents à 50 % du prix de souscription de la période d’engagement restante.

## Formule Utilisateurs actifs mensuels (MAU) {#monthlyactiveusersmauplan}

Vous pouvez choisir une formule MAU comme méthode de facturation préférée. Cette option génère la facturation en fonction du nombre d’utilisateurs actifs uniques par mois. Les utilisateurs actifs uniques par mois sont ajoutés cumulativement pour une période de 12 mois à compter du mois d’activation de la formule. Ce numéro est utilisé pour la facturation de la période.

Utilisez l&#39;exemple suivant pour comprendre comment est calculé le MAU.

Supposons que le nombre d’utilisateurs par mois soit le suivant :

* Mois 1 = 50
* Mois 2 = 500
* Mois 3 = 5 000
* Mois 4 à 12 = 10

Nombre total d’utilisateurs actifs mensuels facturés = Mois 1 + Mois 2 + Mois 3 + Mois 4 à 12 = 50 + 500 + 5 000 + 90 = 5 640.

La facturation pour la période serait de 5 640 utilisateurs.

À la fin de la période de 12 mois, le nombre d’utilisations est remis à zéro et une nouvelle période pour la formule MAU commence. Vous pouvez ajouter plusieurs clés d’activation pour augmenter le nombre de places achetées.

Tout utilisateur qui effectue les actions suivantes ou qui termine son travail en raison d’actions entreprises par d’autres est considéré comme un utilisateur actif unique par mois pour ce mois civil.

* Suivi d’un cours, d’un programme d’apprentissage ou d’une certification.
* Consommation, téléchargement d&#39;une assistance à la tâche ou de pièces jointes à un cours.
* Consommation, téléchargement ou création de notes personnelles.
* Participer à l’apprentissage par les réseaux sociaux en créant des forums, des publications ou des commentaires.
* Achèvement obtenu grâce aux approbations de soumission de certificats externes ou à la participation à des sessions de salle de classe/salle de classe virtuelle.

## Foire aux questions {#frequentlyaskedquestions}

+++Comment ajouter/supprimer des abonnements d’un compte ?

Pour ajouter des abonnements dans un compte, ajoutez le nombre d’utilisateurs pour lesquels vous souhaitez acheter des abonnements. Ensuite, dans le coin supérieur droit, cliquez sur **[!UICONTROL Passer une commande]**. Vérifiez le devis et cliquez sur **[!UICONTROL Continuer]**. Saisissez les détails de votre compte ainsi que ceux de votre carte de crédit. Ensuite, pour acheter les abonnements, cliquez sur **[!UICONTROL Terminer la commande]**.

Pour supprimer un abonnement actif, contactez l’équipe d’assistance Learning Manager.
+++

+++Comment modifier une carte de crédit pour les abonnements ?

Dans le panneau **[!UICONTROL Historique des commandes]** onglet, pour un compte actif, cliquez sur **[!UICONTROL Modifier]**. Dans la page Détails de l’abonnement, cliquez sur **[!UICONTROL Modifier l’abonnement]**. Saisissez les nouvelles informations de votre carte bancaire, puis cliquez sur **[!UICONTROL Mettre à jour le mode de paiement]**.

![](assets/credit-card-details.png)

*Afficher les détails de la carte bancaire*
+++

+++Comment mettre à jour les informations de facturation dans Learning Manager ?

Pour mettre à jour les informations de facturation, procédez comme suit :

1. Se connecter en tant que **Administrateur** et cliquez sur **[!UICONTROL Facturation]**.
1. Dans la liste des commandes, cliquez sur **[!UICONTROL Modifier]**.
1. Dans la page Détails de l’abonnement, cliquez sur **[!UICONTROL Modifier l’abonnement]**.

Choisissez l’élément à modifier :

1. **[!UICONTROL Mode de paiement]:** Utilisez cette option pour mettre à jour les détails de paiement, tels que les données de carte bancaire.
1. **[!UICONTROL Adresse]:** Utilisez cette option pour mettre à jour les détails de l’adresse.
+++

+++ Puis-je annuler partiellement un abonnement ?

Non, vous ne pouvez pas annuler un abonnement partiellement. Si vous devez réduire le nombre de places que vous avez achetées, vous pouvez annuler l’abonnement à la fin du cycle de facturation, puis acheter le nombre de places requises.
+++

+++Comment obtenir une facture pour les paiements par carte de crédit ?

Contact [FastSpring](https://fastspring.com/) pour obtenir une facture pour vos paiements, utilisez l’une des méthodes suivantes :

* Créer une demande de service avec FastSpring à l’aide du lien `https://questionacharge.com`.
* Envoyer un e-mail à FastSpring le `orders@fastspring.com` demande de facture.
