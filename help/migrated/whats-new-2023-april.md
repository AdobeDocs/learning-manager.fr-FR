# Nouveautés de cette version (avril 2023)

## Adobe de l’application Learning Manager pour les Microsofts Teams

La nouvelle application Adobe Learning Manager sur les Microsofts Teams est conçue pour favoriser l’apprentissage dans le flux de travail et stimuler l’apprentissage par les réseaux sociaux. Les élèves pourront accéder au contenu d’apprentissage dans la plateforme de Microsofts Teams sans avoir à basculer sur un navigateur. Veuillez contacter votre CSAM pour obtenir la version bêta de l’application Adobe Learning Manager sur MS Teams.

Pour plus d’informations, voir [Adobe de l’application Learning Manager pour les Microsofts Teams](/help/migrated/adobe-learning-manager-app-microsoft-teams.md).

## Améliorations apportées à l’expérience de formation dirigée par un instructeur

Plusieurs améliorations ont été apportées à l’expérience de formation dirigée par un instructeur (ILT). Les principales améliorations incluent : la possibilité de filtrer les sessions en fonction de l’emplacement, la possibilité de changer d’instance (VILT) sans perdre la progression, un nouvel « Assistant de planification » pour la gestion des conflits dans les sessions de réservation, la possibilité d’associer des « compétences » aux instructeurs et de choisir des instructeurs en fonction des compétences.

### Ce qui a changé

* La page Compétences de l’application d’administration a la possibilité de charger le mappage Compétence-Instructeur.
* La page Compétences contient une nouvelle colonne, Instructeurs. La colonne affiche le nombre d’instructeurs par rapport à la compétence. Si vous cliquez sur le nombre dans la colonne Instructeurs, vous serez redirigé vers une fenêtre contextuelle qui affiche le nombre d’instructeurs affectés à une compétence.

Pour plus d’informations, voir [Attribution de compétences aux instructeurs](/help/migrated/administrators/feature-summary/skills-levels.md#assign-skills-to-instructors).

![CSV de l’instructeur](assets/instructor-csv-new.png)

### Assistant de planification

Gérez les conflits dans les instructeurs de réservation et les classes ou les classes virtuelles. Si vous souhaitez savoir à quelle heure et à quelle date un instructeur est disponible avant de l&#39;affecter au cours, utilisez l&#39;Assistant Planification.

Les auteurs, les administrateurs et les administrateurs/auteurs personnalisés peuvent utiliser l’assistant de planification.

Pour plus d’informations, voir [Assistant de planification](/help/migrated/authors/feature-summary/courses.md#scheduling-assistant).

## Nouvelle application mobile Adobe Learning Manager

Une toute nouvelle application mobile Adobe Learning Manager pour Android et iOS qui offre aux élèves un accès fluide à l’apprentissage lors de leurs déplacements. L’application permet aux élèves d’utiliser la formation sur leurs appareils mobiles et de la reprendre ultérieurement sur leurs ordinateurs de bureau. Grâce à la prise en charge des notifications, des chargements de fichiers, de la facilité de marquage et de partage du contenu d&#39;apprentissage, de l&#39;apprentissage par les réseaux sociaux et plus encore, l&#39;application mobile offre une flexibilité supplémentaire aux élèves et prend en charge l&#39;apprentissage juste-à-temps.

![Nouvelle image de l’interface de mappage mobile](assets/mobile-app.png)

Téléchargez le mobile à partir du Google Play Store et d’Apple App Store. Les utilisateurs de l’ancienne version de l’application seront invités à télécharger et à installer la nouvelle application.

Dans cette version, l’application prend en charge les fonctionnalités suivantes :

La nouvelle application mobile prend en charge les fonctionnalités clés suivantes dans cette version :

* Recherche améliorée, y compris l’affichage des recherches récentes par l’élève et des recherches populaires dans l’organisation dans la zone de saisie de recherche.
* Possibilité de télécharger les relevés de notes des élèves.
* Centre de notifications improvisé : fournit des mises à jour importantes aux élèves sous forme de notifications dans l’application
* Possibilité de trier et de filtrer les objets d’apprentissage dans les pages Catalogue/Mon apprentissage
* Possibilité de mettre en signet des objets d’apprentissage (LO) : les LO marqués d’un signet seront disponibles dans la section « Enregistré par moi » de la page d’accueil de l’élève.
* Prise en charge des programmes d’apprentissage améliorés
* Possibilité de modifier la langue de l’interface mobile en n’importe quelle langue prise en charge par Learning Manager Adobe.
* Prise en charge des chargements de fichiers dans les certifications externes, les sessions CR/VC et les modules d’activité.
* Possibilité d’envoyer des commentaires de cours L1 dans l’application.

* Annonces personnalisées activées par l’administrateur qui s’affichent sous forme de fenêtre contextuelle pour l’utilisateur dans l’application mobile.
* Vous ne pouvez pas ajouter de compétences sur la page des compétences dans l’application.
* Suivi des cours hors ligne.

Pour plus d’informations, voir [Application mobile Adobe Learning Manager](/help/migrated/learners/feature-summary/ipad-android-tablet-users.md).

## Utiliser n&#39;importe quelle application de numérisation de code QR pour numériser les codes QR de Learning Manager

Adobe Learning Manager prend désormais en charge une façon plus intuitive de numériser des codes QR à l’aide de l’application d’appareil photo native sans avoir à télécharger une application qui numérise un code QR.

Pour prendre en charge le workflow précédent, dans lequel vous pouviez numériser un code QR à l’aide du menu de l’application ALM, dans cette version, nous avons fourni des informations sur la façon de passer au nouveau workflow.

Grâce aux améliorations apportées aux workflows basés sur le code QR, les anciens codes QR générés avant cette version ne seront plus pris en charge. Par conséquent, si vous avez généré un code QR avec une ancienne version de Learning Manager pour une session de classe ou de classe virtuelle qui sera effectuée après cette version, vous devez générer un nouveau code QR.

## Autres améliorations

### Recherche d’élève

Les résultats de la recherche sont également classés comme suit :

* Recherches récentes par vous
* Recherches courantes dans votre organisation

La fonctionnalité de recherche a également été améliorée pour les élèves. Les élèves peuvent désormais utiliser des guillemets doubles « ... », plus « + » et moins « - » pour trouver les résultats pertinents plus rapidement et profiter d&#39;une expérience de recherche google.

* Utilisez des guillemets («...») pour rechercher des cours contenant l’expression ou le mot exact. Par exemple, si vous saisissez « science des données », les cours qui commencent par l’expression science des données seront renvoyés.
* Utilisez l’opérateur + pour vous assurer que seuls les résultats contenant l’expression ou le mot spécifique s’affichent. Par exemple, Programmation informatique +python affichera uniquement les cours de programmation informatique contenant le mot « python ».
* Utilisez l’opérateur - pour vous assurer que seuls les résultats ne contenant pas l’expression ou le mot spécifique s’affichent. Par exemple, Programmation informatique -python affichera tous les cours de programmation informatique sauf ceux qui contiennent le mot « python ».

### Fenêtre contextuelle en conflit pour les objets d’apprentissage

La page Présentation affiche une fenêtre contextuelle lors de l’inscription lorsqu’un élève a des sessions en conflit. Si l’inscription à partir de la page de catalogue ou de la page de présentation et qu’un élève a déjà des sessions en conflit, cela déclenchera une fenêtre contextuelle en conflit avec des détails de session en conflit. Même si un élève a une session en conflit, il peut s’inscrire à la nouvelle session.

![avertissement](assets/learner-pop-up.png)

Notez que ce message n’est qu’un avertissement. Vous pouvez toujours vous inscrire à un cours.

### Nouveau modèle

Nous avons ajouté un nouveau modèle d’e-mail appelé Session annulée en raison de la désinscription. Lorsqu’un utilisateur est désinscrit d’une formation, il reçoit l’e-mail d’annulation.

### Nouvelle fonctionnalité de tarification pour les cours

Les administrateurs peuvent désormais définir la devise du compte dans la section Paramètres d’administration > Informations de base. Les administrateurs peuvent spécifier à la fois le symbole et le code devise ISO, par exemple USD, GBP, etc. La valeur par défaut de tous les nouveaux comptes sera $. Cette modification s’applique uniquement aux applications ne faisant pas partie de la formation. Les administrateurs peuvent voir la modification lorsqu’ils prévisualisent le cours en tant qu’élèves. Le symbole monétaire apparaît également dans les relevés de notes des élèves et les rapports de formation.

Pour plus d’informations, voir [Paramètres administrateur](/help/migrated/administrators/feature-summary/settings.md).

### Changer d’instance

Un élève inscrit à une instance particulière d&#39;un cours peut afficher une liste de toutes les instances disponibles du cours et passer à une autre instance qui lui convient le mieux. La raison du changement peut être que l’élève n’a pas assisté à l’instance précédente, ou que la durée de la session pour la nouvelle instance est plus appropriée ou qu’elle peut être différente.

Toute progression effectuée par l’élève dans le cadre du cours, les scores du quiz de l’élève, etc., sont reportés sur la nouvelle instance. Cette fonctionnalité est principalement destinée aux cours en salle de classe et fusionnés, mais nous prendrons en charge tous les types de cours, y compris les cours individualisés.

L’option Afficher toutes les instances s’affiche indépendamment des paramètres de changement d’instance. Les élèves peuvent désormais afficher une option permettant d&#39;afficher toutes les instances s&#39;il existe plusieurs instances d&#39;un cours.

Si l’option Changement d’instance est activée, les élèves peuvent passer à différentes instances jusqu’à ce qu’ils terminent le cours. Si l’option est désactivée, les élèves peuvent afficher les instances, mais ne pourront pas s’inscrire au cours.

Lorsque l’élève sélectionne **Afficher toutes les instances**, l’élève peut voir toutes les instances de cours.

![afficher toutes les instances](assets/view-all-instances-new.png)

Un administrateur peut changer les instances pour les élèves.

![changer d’instance](assets/switch-instances.png)

![changer d’instance sélectionner](assets/switch-instances-select.png)

Vous ne pouvez pas changer d’instance tant qu’une instance du cours n’est pas terminée à partir de l’application Élève ou Administrateur.

Lors de la création d’un cours, les auteurs peuvent basculer entre l’activation ou la désactivation du « changement d’instance ». Le changement d’instance est disponible uniquement pour les cours gratuits.

Pour plus d’informations, voir [Configuration de l’instance](/help/migrated/authors/feature-summary/courses.md).

**Élèves**

Les élèves peuvent passer d&#39;une instance de cours sur liste d&#39;attente à une autre instance. Ils peuvent également changer d’instance de cours dans un parcours d’apprentissage ou une certification.

Après s’être inscrit au parcours d’apprentissage ou à la certification, sur la page du cours, ils peuvent cliquer sur Afficher les instances, puis changer d’instance.

>[!NOTE]
>
>La configuration du commutateur d&#39;instance n&#39;est pas prise en charge pour les types d&#39;inscription nommés par le responsable.
>
>Si vous changez d’instance à partir d’un programme d’apprentissage Flex, la progression sera transférée vers l’autre instance.

Les options sont activées par défaut.

### Sessions d’élèves en conflit

Récupérez la liste de toutes les sessions en conflit pour une instance.

Nous avons ajouté les champs suivants :

* loID
* loInstanceID

**Point de terminaison**: `GET /primeapi/v2/learningObjects/{loId}/instances/loInstanceId/conflictingSessions?page[offset]=0&page[limit]=10`

### Salle de classe en classe virtuelle

Exécutez une recherche basée sur l&#39;emplacement pour les cours VC. Un nouvel attribut roomLocation est présent dans le modèle de ressource pour indiquer l&#39;emplacement de forme libre fourni lors de la création d&#39;un cours VC.

Nous avons apporté les modifications suivantes :

**LO**

Un nouveau paramètre de requête, filter.loFormat=Virtual Classroom pour l’API learningObjects.

**Point de terminaison**: GET /primeapi/v2/learningObjects

**Calendrier**

Un nouveau paramètre de requête, filter.allSessions=false pour l&#39;API de calendrier. La valeur par défaut du paramètre est false. Si la valeur est True, l’API renvoie toutes les sessions de calendrier d’un élève.

**Point de terminaison**: `GET /primeapi/v2/users/<id>/calendar?filter.allSessions=false`

### Historique de recherche des élèves

**Search**

Un nouveau paramètre de requête, persistSearchHistory. La valeur par défaut est true, ce qui rend persistante la requête pour les suggestions de recherche.

**Point de terminaison**: GET /primeapi/v2/search?persistSearchHistory=true

**Suggestions**

Un nouveau paramètre de requête, suggestionType. Les valeurs acceptées sont les suivantes :

* learnerHistory (par défaut)
* accountHistory

**Point de terminaison**: GET /primeapi/v2/search/suggestions/?suggestionType=learnerHistory

### Filtrage de groupe d’utilisateurs

Les API d’objet d’apprentissage et de recherche fourniront des filtres pour récupérer tous les objets d’apprentissage qui appartiennent au groupe d’utilisateurs via les filtres spécifiés. Les API prennent en charge les filtres sous forme de liste séparée par des virgules.

Nous avons fourni un nouveau filtre, filter.lo.announcementGroups, pour récupérer tous les objets d’apprentissage qui appartiennent aux filtres de groupe d’utilisateurs fournis.

Il peut s’agir d’une liste à plusieurs valeurs séparées par des virgules pour plus de flexibilité, et le traitement serait basé sur l’opération « OU » entre les différents groupes. par exemple, obtenez tous les objets d’apprentissage qui relèvent des groupes d’utilisateurs donnés.

### Filtrage du groupe d&#39;utilisateurs des annonces pour les pertes dans l&#39;application Élève
